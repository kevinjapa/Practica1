Que son los Estandares Web

Los estándares web fueron creados para proteger el ecosistema web y mantenerlo accesible para todos. Antes, era necesario construir sitios web que se adaptaran a navegadores específicos. Con la introducción de los estándares, se alentó a los creadores de navegadores a adherirse a una forma estandarizada de hacer las cosas, lo que permitió una compatibilidad cruzada más fácil y eliminó la necesidad de crear múltiples versiones del mismo sitio web.

Algunos de los estándares Web más conocidos y ampliamente utilizados son:

HTML (HyperText Markup Language), para definir la estructura de los documentos.
XML (eXtensible Markup Language), que sirve de base para un gran número de tecnologías.
CSS (Cascading Style Sheets), que permite asignar estilos para la representación de los documentos.
Javascript, que permite otorgar dinamismo y funcionalidad.

Como fueron Creados

En la creación de estándares web, se sigue un proceso controlado que consta de varias etapas para asegurar la calidad de la especificación. Este proceso permite la intervención de todos los usuarios para mejorar los documentos. Tras la revisión de especialistas en la materia, se obtienen estándares de calidad que se depuran exhaustivamente antes de ser considerados como recomendación o estándar para su disponibilidad global.

Uso Habitual

Un sitio web que cumple con los estándares web tiene partes de código HTML, CSS, XML y JavaScript válido o casi válido, y la parte HTML cumple ciertas guías de accesibilidad y semántica. Para cumplir con los estándares web, se deben seguir algunas publicaciones fundamentales, como:

1. Recomendaciones para lenguajes de marcado, como HTML, XHTML, SVG y XForms, de W3C.
2. Recomendaciones para hojas de estilo, especialmente CSS, de W3C.
3. Estándares para ECMAScript, más comúnmente conocido como JavaScript, de Ecma International.
4. Recomendaciones para Document Object Models (DOM), de W3C.
5. Nombres y direcciones de página correctamente formados y recursos referenciados, como URIs, basado en RFC 2396, de IETF.
6. El uso apropiado de los protocolos HTTP y MIME para desplegar la página, retornar datos, pedir recursos referenciados, basado en RFC 2616, de IETF.
